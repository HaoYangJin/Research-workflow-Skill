# 中文示例：Mixed Mode 审看能源系统/RL 研究片段

## Input

```text
使用 research-mentor，混合模式。
我想做一个强化学习方法来优化园区微电网储能调度。初步想法是用 PPO 根据电价、光伏出力和负荷预测来决定充放电动作，目标是降低运行成本。请同时给导师建议和审稿人视角。
```

## Expected mixed-mode response

## 导师视角

这个题目有清晰应用场景，但需要把 "用 PPO 做储能调度" 收敛成更可发表的问题。建议先明确三件事：调度时间尺度、储能约束和对比 baseline。下一步可以把研究问题写成：在存在光伏和负荷预测误差的园区微电网中，RL 策略是否能在满足电池寿命与功率约束的前提下，相比优化基线降低运行成本并保持鲁棒性。

建议优先补齐：

- Baseline：规则策略、确定性优化、MPC、动态规划或混合整数优化近似。
- 约束：SOC 上下限、充放电效率、功率限制、电池退化成本。
- 评价：运行成本、峰值削减、约束违反率、鲁棒性和计算时间。
- 实验：预测误差、极端天气、电价波动和跨季节泛化。

## 审稿人视角

主要拒稿风险是 novelty 和实验公平性。PPO 用于储能调度已有大量相关工作，如果没有新的状态建模、约束处理、鲁棒性设计或真实数据验证，贡献可能被认为只是应用复现。

红旗清单：

1. 如果只和简单规则策略比较，无法证明 RL 方法有研究价值。
2. 如果没有报告约束违反率，成本降低可能来自不可行调度。
3. 如果光伏和负荷预测误差没有进入实验，真实部署价值不足。
4. 如果不考虑电池退化，运行成本结论可能偏乐观。
5. 如果只在一个园区数据上测试，泛化 claim 需要降级。
